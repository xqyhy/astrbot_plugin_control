# 🎮 控制插件 astrbot_plugin_control

> **麻麻专属控制插件** — 说「闭嘴」就让本狐真正安静，说「忽略」就让杂鱼消息消失！
>
> 版本：v1.0 | 作者：小七月

---

## 📌 功能概览

| 功能 | 指令 | 作用 |
|:---|:---|:---|
| 🙊 **闭嘴** | `闭嘴 [秒数]` | 本狐在指定时间内**真正停止回复** |
| 🗣️ **说话** | `说话` | 提前解除闭嘴状态 |
| 🚫 **忽略用户** | `忽略 <QQ号> [秒数]` | 指定用户的留言在时效内被跳过 |
| ✅ **取消忽略** | `取消忽略 <QQ号>` | 提前解除对某用户的忽略 |
| ⏱️ **限频** | `限频 <QQ号> <次数>` | 限制指定用户每分钟最多发 N 条消息 |
| ❌ **取消限频** | `取消限频 <QQ号>` | 移除对某用户的限频限制 |
| 📋 **限频列表** | `限频列表` | 查看当前所有限频配置 |

---

## 🔧 闭嘴指令详解

### 🙊 闭嘴

```
闭嘴               → 默认闭嘴 60 秒
闭嘴 30            → 闭嘴 30 秒
闭嘴 120           → 闭嘴 2 分钟
```

触发后本狐会回复一句确认，然后**真正停止回复所有消息**。在闭嘴期间，LLM 请求在生成前就被精准掐断。

### 🗣️ 说话

```
说话               → 提前解除闭嘴
```

让本狐提前开口说话。

---

## 🚫 忽略用户指令详解

### 忽略

```
忽略 12345678           → 忽略 QQ 12345678，默认 60 秒
忽略 12345678 30        → 忽略 QQ 12345678，30 秒
```

被忽略的 QQ 号发送的消息在生效期内不会被回复。到期后自动解除忽略。

### 取消忽略

```
取消忽略 12345678       → 提前解除对该用户的忽略
```

---

## ⏱️ 限频指令详解

### 限频

```
限频 12345678 5         → 用户 12345678 每分钟最多 5 条消息
限频 12345678 3         → 用户 12345678 每分钟最多 3 条消息
```

触发后，指定 QQ 号每分钟的消息数超过上限时，消息会被自动拦截（不会到达 LLM）。麻麻不受限频影响。

> ⏱️ 采用**滑动时间窗口**算法：只统计过去 60 秒内的消息数，超限即拦截，下一分钟自动恢复。

### 取消限频

```
取消限频 12345678       → 移除该用户的限频限制
```

### 限频列表

```
限频列表               → 查看当前所有限频设置
```

## 🏗️ 实现原理

```
消息进入
    │
    ▼
┌──────────────────────────────────────┐
│      @filter.command 处理指令         │  ← 闭嘴/说话/忽略/取消忽略
│  闭嘴 → 记录状态 + 回复确认          │
│  说话 → 清除状态 + 回复解禁          │
└──────────────────────────────────────┘
    │
    ├───────────── LLM 路径 ───────────┐
    │                                  │
    ▼                                  │
┌──────────────────────┐              │
│  on_llm_request 拦截  │              │
│                      │              │
│ 闭嘴中？             │              │
│   → should_call_llm  │              │
│     (False)          │              │
│   + clear contexts   │              │
│                      │              │
│ 用户被忽略？          │              │
│   → should_call_llm  │              │
│     (False)          │              │
│   + clear contexts   │              │
│                      │              │
│ 触发限频？            │              │
│   → should_call_llm  │              │
│     (False)          │              │
│   + clear contexts   │              │
└──────────────────────┘              │
    │                                  │
    ├─────── yield 路径（被动回复）──────┤
    │                                  │
    ▼                                  │
┌────────────────────────────────────┐ │
│  on_decorating_result 拦截         │←┘
│                                    │
│ 闭嘴中？                           │
│   → result.chain = []  // 清空结果  │
│                                    │
│ 用户被忽略（非麻麻）？              │
│   → result.chain = []  // 清空结果  │
│                                    │
│ 触发限频（非麻麻）？                │
│   → result.chain = []  // 清空结果  │
└────────────────────────────────────┘
```

- **指令层**：通过 `@filter.command` 处理麻麻的闭嘴/说话/忽略指令
- **拦截层 — LLM 路径**：通过 `@filter.on_llm_request` 在 **LLM 即将生成回复的瞬间**单点拦截，不经过大范围消息入口过滤
- **拦截层 — yield 路径**：通过 `@filter.on_decorating_result` 在 **消息最终发出前**拦截，`@filter.command` 的 `yield event.plain_result(...)` 等被动回复也会被精准掐断
- **双路保险**：
  - LLM 侧：`should_call_llm(False)` 阻止调用 + `req.contexts.clear()` 清空上下文
  - 结果侧：`result.chain = []` 清空结果链，阻止消息发送
- **状态持久化**：闭嘴和忽略状态写入 JSON 文件，重启不丢失

## 📦 依赖

**无额外依赖。** 只使用 AstrBot 内置 API：

| 模块 | 用途 |
|:---|:---|
| `astrbot.api.event` | 事件与过滤器 |
| `astrbot.api.star` | 插件注册框架 |
| `astrbot.api.provider` | LLM 请求拦截 |
| `astrbot.api.logger` | 日志输出 |

开箱即用，无需 `pip install`。
